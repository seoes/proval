---
title: "Why self-hosted code review"
description: "Teams that want control over code, models, and cost."
date: 2026-06-03
---

Many AI review tools are convenient SaaS products. They work until your team needs to keep diffs and review context off third-party servers, use GitLab or Forgejo as the primary host, or run a model you already operate internally.

Self-hosted review is not about rejecting cloud tools entirely. It is about a team layer that stays on your network: webhooks in, comments out, policies you control, and models you choose.

Proval is built for that layer. Set it up once, connect repositories, and let reviews run when events arrive instead of waiting for someone to paste a diff into a chat window.
